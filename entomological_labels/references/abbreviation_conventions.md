## Collector Name Conventions

### Formatting Rules

1. **Single collector:** First initial(s) + surname: `B.A.S. de Medeiros`
2. **Two collectors:** `B. de Medeiros & J. Smith` — use `&` not "and"
3. **Three+ collectors:** `B. de Medeiros et al.` — use first collector only
4. **Team/expedition names:** Can abbreviate: `FMNH Expedition` → `FMNH Exped.`

### Initial Rules

- Use first and middle initials with periods: `B.A.S.`
- For compound surnames, keep the full surname: `de Medeiros`, `van der Berg`
- For very long names, can drop middle initials: `B. de Medeiros` instead of `B.A.S. de Medeiros`

### Common Abbreviation Patterns

| Full | Abbreviated | When |
|------|------------|------|
| B.A.S. de Medeiros, J. Smith, L. Jones | B.A.S. de Medeiros et al. | > 2 collectors |
| B.A.S. de Medeiros & J. Smith | B. de Medeiros & J. Smith | if still too long |
| Field Museum expedition team | FMNH team | institutional teams |

## Date Formatting

### Roman Month Numerals

| Month | Numeral |
|-------|---------|
| January | i |
| February | ii |
| March | iii |
| April | iv |
| May | v |
| June | vi |
| July | vii |
| August | viii |
| September | ix |
| October | x |
| November | xi |
| December | xii |

### Format Examples

| Scenario | Format | Example |
|----------|--------|---------|
| Single date | day.month.year | 20.v.2012 |
| Date range, same month | day-day.month.year | 5-12.viii.2024 |
| Date range, diff months | day.month-day.month.year | 20.v-3.vi.2012 |
| Date range, diff years | day.month.year-day.month.year | 28.xii.2011-5.i.2012 |
| Month + year only | month.year | viii.2024 |
| Year only | year | 2024 |

### Rules
- No leading zeros on days: `5.viii.2024` not `05.viii.2024`
- Roman numerals always lowercase
- No spaces around periods or hyphens in dates
- Year always 4 digits

## Taxonomic Abbreviations

### Formatting Rules

| Element | Style | Example |
|---------|-------|---------|
| Genus name | **italic** | *Anchylorhynchus* |
| Specific epithet | **italic** | *trapezicollis* |
| Subspecific epithet | **italic** | *trapezicollis* |
| "sp." (undetermined) | roman | *Anchylorhynchus* sp. |
| "spp." (multiple species) | roman | *Anchylorhynchus* spp. |
| "cf." (compare with) | roman | *Anchylorhynchus* cf. *trapezicollis* |
| "aff." (affinity with) | roman | *Anchylorhynchus* aff. *trapezicollis* |
| "nr." (near) | roman | *Anchylorhynchus* nr. *trapezicollis* |
| "n. sp." (new species) | roman | *Anchylorhynchus* n. sp. |
| "n. gen." (new genus) | roman | *Anchylorhynchus* n. gen. |
| Authority name | roman | Vaurie, 1954 |
| Family name | roman | Curculionidae |
| "det." (determined by) | roman | det. B. de Medeiros 2024 |
| Male symbol | roman | Unicode: ♂ (U+2642) |
| Female symbol | roman | Unicode: ♀ (U+2640) |

### "cf." and "aff." Placement

When `cf.` or `aff.` is present, the formatting is:

```
*Genus* cf. *species*    ← genus italic, cf. roman, species italic
*Genus* aff. *species*   ← genus italic, aff. roman, species italic
```

The qualifier goes between genus and species, all in the correct style.

### "sp." Variants

All of these normalize to "sp." on labels:
- sp., sp.1, sp.2, sp. nov., sp.n., sp. A, sp. B, sp. indet.

Display as: `*Genus* sp.` — genus italic, "sp." roman

### Determiner Line

Format: `F. Last det. YEAR`

- Use initials + surname for determiner
- "det." is always lowercase, roman
- Year is 4 digits
- If no year: `F. Last det.`

## Host Plant Formatting

### Standard Format

```
On *Genus species* Authority    ← "On" roman, genus+species italic, authority roman
```

### Abbreviation Rules

- "On" is always roman, followed by a space
- Plant genus + species in italic
- Plant authority in roman (often omitted on small labels if space is tight)
- If only genus known: `On *Genus* sp.` — genus italic, "sp." roman

### When Host Plant Text Is Too Long

1. Drop the plant authority first
2. Abbreviate genus to initial: `On *G. species*`
3. If still too long, ask the user

## Elevation Formatting

| Scenario | Format | Example |
|----------|--------|---------|
| Single elevation | value + "m" | 1200m |
| Elevation range | min-max + "m" | 800-1200m |
| Approximate | ca. + value + "m" | ca. 1500m |

- No space between number and "m"
- No leading zeros
- Round to nearest meter (no decimals)

## Coordinate Formatting

| Scenario | Format | Example |
|----------|--------|---------|
| Latitude (south) | S + absolute value | S0.12345 |
| Latitude (north) | N + absolute value | N45.67890 |
| Longitude (west) | W + absolute value | W35.67890 |
| Longitude (east) | E + absolute value | E120.12345 |

- 5 decimal places
- Prefix with hemisphere letter (no degree symbol)
- Separate lat and lon with comma+space: `S0.12345, W35.67890`

## Batch Abbreviation Strategy

When processing labels, group identical values and abbreviate in batches:

1. **Count unique values** for each field
2. **Check character limits** (~28-30 chars at 4pt)
3. **For values exceeding the limit**, present to user grouped by frequency:
   - "45 specimens have locality 'Parque Nacional da Gorongosa' (36 chars) — abbreviate to 'Gorongosa NP' (12 chars)?"
   - "120 specimens have collector 'B.A.S. de Medeiros, J.R. Smith, L.K. Jones' (43 chars) — abbreviate to 'B. de Medeiros et al.' (21 chars)?"
4. **Apply standard abbreviations** from this file as suggestions
5. **Let the user decide** — never auto-abbreviate without confirmation
